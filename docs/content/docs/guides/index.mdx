# Integration Guides

These guides walk you through complete integration scenarios, from signup to first sale.

## The Complete Picture

Tuish is a **fully integrated monetization stack**. The SDK, API, and infrastructure work together out of the box:

```
┌─────────────────────────────────────────────────────────────────┐
```

## How It All Connects

**Developer Setup:**
1. Sign up via CLI (`tuish signup`)
2. Connect Stripe account (`tuish connect`)
3. Create product (`tuish products create`)
4. Integrate SDK into your app

**Customer Purchase (Browser):**
1. SDK checks for license → none found
2. SDK creates checkout session → API → Stripe
3. Customer pays in browser
4. SDK polls for completion → receives signed license
5. License cached locally → works offline

**Customer Purchase (Terminal):**
1. SDK checks for license → none found
2. Customer enters email → OTP sent via SMS
3. Customer verifies OTP → receives identity token
4. SDK fetches saved cards → customer selects one
5. Purchase OTP sent → customer confirms
6. Payment processed → license issued
7. License cached locally → works offline

All of this is **already implemented** and ready to use.
